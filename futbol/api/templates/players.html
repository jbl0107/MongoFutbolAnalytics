Información sobre este Documento

En la documentación de swagger (situada en la url 127.0.0.1/8000/docs/) no pueden realizarse operaciones POST. Si desea probar al completo la API, pongase en contacto con nosotros a través del correo que se muestra en la documentación comentada anteriormente (en la sección "Contact the developer") para recibir una invitación al workspace de Postman donde todas las peticiones pueden ser probadas. Para probar la peticion POST de este Documento en Postman, deberá seleccionar la opción raw-JSON de Body en el caso de que no vaya a asociarle ninguna foto al jugador. Si va a añadirle una foto, deberá elegir la opción form-data del apartado Body. En este último caso, asegúrese de que el atributo "photo" sea de tipo "File", para poder seleccionar un archivo de su ordenador.

Cuando en una URL se indica que debe especificar el nombre de una liga, tipo de gol o competición, y la que desea consultar tiene espacios, debe sustituir estos por "-". Un ejemplo: si desea buscar información sobre la competición "Premier League", en la URL deberá aparecer Premier-League.

Objeto JSON

Este es un ejemplo del JSON de un objeto del Documento Player. Todos los atributos son obligatorios excepto "photo" y "gamesPlayed".
• Si no especifica el atributo "gamesPlayed", se le asignará automáticamente un valor de 0 para este atributo.
• Puede no especificar el atributo "photo" al registrar un jugador, en cuyo caso tendrá un valor de "None" cuando lo obtenga mediante una peticion GET, como el que se muestra de ejemplo.
• El atributo "name" es único. Su longitud máxima es de 100 caracteres.
• El conjunto de atributos "team" y "dorsal" es único.
• El atributo "team" hace referencia a un objeto del Documento Team.
• El atributo "goodLeg" solo puede tomar estos valores:

  • Zurdo
  • Diestro
  • Ambidiestro
